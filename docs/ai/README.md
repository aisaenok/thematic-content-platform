# AI-документация проекта

Этот раздел описывает, как использовать AI-ассистентов и Codex при разработке проекта **Thematic Content Platform**.

Проект является портфолио-прототипом wiki/content-платформы для тематических энциклопедических сайтов. Он создается как greenfield-модернизация старого проекта Alien Wiki https://github.com/FeHHix/alien-wiki, но не является прямой миграцией старой кодовой базы один к одному.

## Зачем нужны эти документы

AI-ассистенты полезны для ускорения разработки, но без контекста они часто:

- добавляют лишние зависимости;
- делают слишком широкие рефакторинги;
- смешивают domain-логику и UI;
- усложняют архитектуру раньше времени;
- предлагают backend/CMS/Auth до того, как они реально нужны;
- генерируют код, который выглядит рабочим, но плохо ложится в архитектуру проекта.

Эти документы задают рамки, чтобы AI помогал точечно и предсказуемо.

## Состав раздела

```text
docs/ai/
  README.md
  operating-model.md
  project-map.md
  task-brief-template.md
```

### `operating-model.md`

Правила работы с AI-ассистентами:

- какие задачи можно отдавать Codex;
- какие задачи нельзя делать без отдельного архитектурного решения;
- как формулировать ограничения;
- какой результат считать готовым;
- что обязательно проверять после генерации кода.

### `project-map.md`

Карта проекта:

- назначение проекта;
- целевая архитектура;
- предполагаемая структура монорепозитория;
- границы между приложениями, пакетами, domain-логикой и UI;
- решения, которые уже приняты;
- решения, которые пока откладываются.

### `task-brief-template.md`

Шаблон задачи для Codex.

Используй его перед тем, как просить Codex что-то сгенерировать или изменить. Чем точнее задача, тем меньше риск получить лишний код или сломать архитектуру.

## Базовый принцип

AI не должен сам принимать крупные архитектурные решения.

Codex можно использовать как помощника для маленьких и хорошо ограниченных задач:

- создать типы;
- добавить компонент;
- реализовать адаптер;
- написать mock data;
- обновить README;
- добавить простую страницу;
- исправить локальную ошибку;
- подготовить тест для уже существующей логики.

Но Codex не должен без явного задания:

- менять стек;
- добавлять Redux, MobX, Redux-Saga или другой state manager;
- добавлять backend;
- добавлять CMS;
- переписывать структуру монорепозитория;
- делать cross-cutting refactoring;
- менять архитектурные границы между слоями.

## Контекст проекта в одном абзаце

**Thematic Content Platform** — это портфолио-прототип wiki/content-платформы для тематических энциклопедических сайтов. Проект переосмысляет старый Alien Wiki как современное приложение на базе Nx, Next.js App Router, TypeScript, typed content model, server-side rendering и подключаемого content source layer. Вселенная Alien используется как демонстрационная предметная область, но архитектура должна оставаться достаточно общей, чтобы на ее базе можно было сделать сайт под другой тематический домен.

## Как использовать с Codex

Перед задачей желательно приложить к промпту:

1. краткое описание задачи;
2. ссылки на релевантные файлы;
3. ограничения;
4. ожидаемый результат;
5. список того, что нельзя менять;
6. команду проверки, если она уже есть в проекте.

Пример:

```text
Задача:
Создать typed domain model для article-сущности.

Контекст:
Проект — Nx + Next.js portfolio-grade wiki/content platform.
Domain-типы должны лежать отдельно от UI.

Файлы:
- packages/content-domain/src/article.ts
- packages/content-domain/src/index.ts

Ограничения:
- не добавлять runtime-зависимости;
- не добавлять CMS-интеграцию;
- не добавлять UI;
- не менять структуру apps/web.

Ожидаемый результат:
- тип Article;
- тип ArticleSummary;
- тип ContentRelation;
- тип Slug;
- экспорт из index.ts.
```

## Важное правило

Если задача звучит как “сделай всю платформу”, ее нужно разбить на маленькие вертикальные шаги.

Правильный фокус на старте:

```text
content model → mock content source → Next server page → UI blocks → SEO metadata
```

Неправильный фокус на старте:

```text
CMS + backend + auth + admin panel + search + deploy + full design system
```
